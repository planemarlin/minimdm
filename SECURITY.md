# Security Policy

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

The preferred way to report a security issue is via [GitHub's private vulnerability reporting](https://github.com/planemarlin/minimdm/security/advisories/new). This keeps the details confidential until a fix is available.

If you are unable to use that channel, you can reach the maintainer by email at **minimdmsecurity@gmail.com**. Include as much detail as possible: a description of the issue, steps to reproduce, the potential impact, and any suggested mitigations.

## Response Time

We aim to respond to all security reports within **5 days**.

## Supported Versions

Only the latest released version of miniMDM receives security fixes. If you are running an older version, please upgrade before reporting.

## Disclosure Policy

We follow a coordinated disclosure process. Once a fix is ready and released, the vulnerability will be publicly disclosed via a GitHub Security Advisory. We ask that reporters refrain from public disclosure until that advisory is published.
