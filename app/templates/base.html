miniMDM Quick Reference

Records

Create, edit, and delete records from the object list. Deleted records are soft-deleted — they are hidden from the list but not removed from the database. Use Show deleted to view them; their history page allows recovery via Revert.

History & Revert

Every change to a record is saved as a new version. Click History on any record to see all versions, then Revert to restore a previous one. Reverts are themselves versioned and audited.

Reason for change

The optional Reason field on create, edit, and delete forms is stored in the audit log alongside the change. Use it to explain why a value was changed.

Import & Upsert

Import records from CSV, TSV, or JSON. Insert only always adds a new record. Upsert on <attribute> updates existing records whose value matches on that attribute and inserts the rest.

Audit log

All data changes and authentication events (login, logout, user management) are recorded in the audit log, available from the header navigation (admin only).

Access control

Admins have full access. Non-admin users have no access by default — an admin must grant read and/or write access per schema from the User Management page.
