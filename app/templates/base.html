miniMDM Quick Reference

Records

Create, edit, and delete records from the object list. Deleted records are soft-deleted — they are hidden from the list but not removed from the database. Use Show deleted to view them; their history page allows recovery via Revert.

Lifecycle states

Every record has a state: Active · Master (the golden record — the authoritative version), Draft (a pending change or new record awaiting approval), or Retired (no longer in use). Editing an active record creates a draft copy alongside it — the active record is unchanged until a Publisher approves. Use the state filter on the list page to switch between views.

Publish promotes a draft to the active golden record. Retire transitions an active record to retired. Both require Publisher or Admin role and are available on the record detail page. Depending on the object, some of these actions may be restricted by your administrator.

Source & provenance

Records can carry optional provenance metadata: source system (e.g. ERP, CRM) and a source ID linking back to the originating system's identifier. When set, these are shown in the record detail view and preserved across all history snapshots.

History & Revert

Every change to a record is saved as a new version. Click History on any record to see all versions, then Revert to restore a previous one. Reverts are themselves versioned and audited.

Reason for change

The Reason field on create, edit, and delete forms is stored in the audit log alongside the change. Use it to explain why a value was changed. For some objects the field is mandatory — the form will indicate this with a red asterisk.

Import & Export

Import records from CSV, TSV, or JSON using the Import option in the Tools menu on the list page. Insert only always adds a new record. Upsert on <attribute> updates existing records that match on that attribute and inserts the rest. Depending on the object, importing directly as active may require Publisher role or may not be permitted at all.

Export the current view as CSV, TSV, or JSON using the Tools menu.

Audit log

All data changes and authentication events (login, logout, user management) are recorded in the audit log, available from the header navigation (admin only). Filter by schema, object, action, or username.

Access control

Four roles are supported, each building on the previous: Viewer (read-only), Editor (create, update, delete, import as draft), Publisher (additionally: publish drafts, retire records, import as active), and Admin (full access across all schemas). Roles are granted per schema from the User Management page.
